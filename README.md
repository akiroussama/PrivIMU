> **Can anonymous IMU signals identify you?** PrivIMU is a reproducible IoT-security project showing how smartphone accelerometer and gyroscope traces can become behavioral quasi-identifiers.

## Why this project matters

Smartphones, smartwatches, health trackers, sport apps, and many IoT systems collect motion data. These traces are often treated as *anonymous* because they do not contain a name, email address, phone number, or face. PrivIMU tests a stronger question:

> **If an attacker only sees inertial signals, can they still infer which enrolled user produced them?**

The project uses the public [MotionSense dataset](https://github.com/mmalekzadeh/motion-sense), which contains iPhone motion data from 24 participants performing six activities. The final goal is not to attack real people; it is to demonstrate, measure, and discuss privacy leakage in a controlled academic setting.

## Quickstart

```bash
# 1) Clone your repository
# git clone https://github.com/akiroussama/PrivIMU.git
# cd PrivIMU

# 2) Create an environment
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
python -m pip install --upgrade pip
python -m pip install -e ".[dev,app]"

# 3) Run tests
pytest -q

# 4) Download MotionSense
python data/download.py --dest data/raw/motionsense

# 5) Train the Random Forest baseline
python -m privimu.train \
  --data-root data/raw/motionsense \
  --model rf \
  --output-dir . \
  --window-size 50 \
  --step-size 25 \
  --n-splits 5

# 6) Launch the live demo
streamlit run streamlit_app.py
```

### Windows PowerShell shortcut

GNU `make` is not installed by default on Windows. Use the included PowerShell task runner instead:

## Evaluation design

PrivIMU is a **closed-set identity re-identification** experiment. The enrolled identities are the 24 MotionSense subjects. The model receives motion windows and predicts the most likely subject ID.

Default protocol:

- dataset: MotionSense `A_DeviceMotion_data`, iPhone in-pocket;
- sensors: `rotationRate.{x,y,z}` + `userAcceleration.{x,y,z}`;
- windowing: 1 second = 50 samples;
- overlap: 50% = 25-sample step;
- normalization: z-score per window and per channel;
- features: 60 statistical/spectral descriptors per window;
- split: group-aware cross-validation by activity/trial group;
- metrics: top-1, top-3, F1-macro, confusion matrix, per-subject F1, latency, entropy leakage.

## Demo story

The live demo should make the privacy risk intuitive:

```text
Anonymous IMU signal → model posterior → Top-3 likely identities → entropy leakage score
                                            │
                                            ▼
                          Add Gaussian noise → confidence drops
```

This is a defensive academic demonstration. It must not be used to identify real people outside the public dataset.

