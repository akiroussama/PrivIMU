# Streamlit WOW features

This update upgrades `streamlit_app.py` from a simple classifier demo into a small privacy lab.

## WOW #1 — Live identity-lock replay

The app now computes one posterior distribution per sliding IMU window and shows when the identity becomes stable enough to be called an "identity lock". This makes the attack understandable during a live presentation: the audience sees confidence rising over time instead of only seeing a static top-3 table.

## WOW #2 — Blue-team defense lab

The sidebar includes a defense sweep. The app runs multiple Gaussian-noise levels and visualizes the privacy/utility trade-off using:

- top-1 identity confidence,
- privacy leakage ΔH in bits,
- a signal-preservation proxy.

This turns the demo from "we can attack" into "we can measure and discuss mitigation".

## WOW #3 — One-click evidence card

The app can generate a downloadable JSON and Markdown evidence card containing:

- input source,
- selected noise level,
- top-3 identity posterior,
- privacy entropy/leakage,
- identity-lock status,
- defense-sweep recommendation.

This is useful for the professor because the demo output becomes an artifact, not just a screen impression.

## Demo script

1. Start with no noise: `σ=0.00`.
2. Show the anonymous IMU signal.
3. Show the top-3 identity posterior.
4. Scroll to the identity-lock replay and explain: "the attack becomes stable after a few windows".
5. Enable the blue-team sweep and show how noise reduces confidence/leakage.
6. Download the evidence card and say: "the live demo produces an audit artifact".

## Important

If `models/rf.joblib` is missing, the app runs in fallback UI mode with synthetic posteriors. This is only for deployment testing. Final scientific claims must come from `reports/metrics.json` generated by the training/evaluation pipeline.

## Demo CSV source update

The upload/download helper now uses the committed MotionSense walking samples under:

```text
examples/wlk_7/sub_*.csv
```

The Streamlit sidebar lists only the raw links for files that exist in the repository checkout, so the app no longer points to the old broken `demo/sample_motionsense_like.csv` URL. The default demo path is now a real MotionSense-style CSV from `examples/wlk_7` when available.
