Topic

PrivIMU — Re-identification Risk in Anonymous Motion Sensor Traces. We investigate a privacy gap that is rarely addressed in IoT systems: inertial signals (accelerometer + gyroscope) collected by smartphones and wearables, although they contain no direct identifier (no name, e-mail, MAC, IMEI), still carry enough behavioural signature to re-identify a user among a known candidate set.

Our approach: reproducibility and methodological honesty (our key differentiator)

We deliberately built a verifiable scientific mini-laboratory rather than a decorative project. Four guarantees underpin every number we report:

  1. No invented numbers. Every metric in the report and slides is regenerated by the code into reports/metrics.json. If a value is not in that file, it is not in the report. We applied this rule without exception.
  2. Honest cross-validation. We use 5-fold GroupKFold with the group key set to activity_trial. A naive K-fold mixing windows of the same trial across train and test would inflate accuracy through within-trial leakage. Our reported numbers are measured under a protocol where the model never sees the test trial during training — this is what makes the result defensible.
  3. Tested and CI-validated pipeline. Structured Python package (privimu.{data, features, model_rf, train, evaluate, viz, metrics}), 8 pytest unit tests, GitHub Actions workflow on every push.
  4. Reproducible in 4 commands. From a clean clone: pip install → download.py → privimu.train → streamlit run. Runs locally on Windows, Linux, macOS, and equally on Colab with no setup.

Results — what the numbers actually mean

Trained on the public MotionSense dataset (24 subjects, 6 activities, iPhone DeviceMotion at 50 Hz, 55 982 windows of 1 second), our Random Forest classifier (200 trees) achieves the following out-of-fold metrics:

Metric Value Random baseline Lift What it means
Top-1 accuracy 39.06 % 4.17 % (1/24) 9.4× The right subject is the model's first guess on 39 % of one-second windows — almost ten times more often than chance.
Top-3 accuracy 52.80 % 12.50 % (3/24) 4.2× The right subject is among the top 3 ranked candidates more than half the time. This is the privacy-relevant metric: an attacker rarely needs certainty, just a short list.
Macro-F1 0.391 0.042 9.3× Averaged equally across the 24 subjects — no hiding poor users behind class imbalance.
Privacy leakage ΔH 0.700 bit / window 0 bit — On average, each one-second window leaks 0.7 bit of identity information out of a theoretical maximum of log2(24) ≈ 4.59 bit (about 15 % of the maximum, per single window).
Inference latency 42 ms / window — — Fast enough for real-time identification on a CPU.

The headline message is not that 39 % constitutes a deployable identification system — it is that an off-the-shelf classifier on a modest 24-subject benchmark already achieves an order-of-magnitude lift over chance, with every explicit identifier removed. Under larger populations, deeper models, longer observation windows, or correlation with device-fingerprint signals, this number can only grow. The right reaction is to treat motion data as personal data before the attack scales.

Two interactive artefacts (please test them before reading the report)

Beyond the PDF and PPTX, we deployed two resources that we strongly encourage you to open:

🔗 Live Streamlit demo

https://privimu.streamlit.app/

Fully deployed interactive web application. On opening, you can either select one of the bundled MotionSense walking samples (examples/wlk_7/sub_*.csv) or upload any MotionSense-like CSV. The app then computes and displays:

  1. Plotly visualisation of the six IMU channels (rotationRate.{x,y,z} + userAcceleration.{x,y,z}).
  2. Top-3 ranked identities with posterior probabilities — the "ranked attack" output.
  3. Window-by-window identity-lock replay with a confidence curve and the timestamp at which the model's top-1 hypothesis stabilises.
  4. Privacy entropy leakage ΔH in bits, computed live from the posterior.
  5. Gaussian-noise defence slider + automatic σ sweep — visualises the privacy / utility trade-off (top-1 confidence vs leakage vs signal preservation) and recommends a defence noise level.
  6. Evidence card export — one-click download of a JSON + Markdown bundle (source, top-3, ΔH, σ, lock-time, timestamp) suitable for a security review.

You can test the entire attack → defence loop on any CSV in less than a minute.

🔗 Google Colab notebook — one-click reproduction

https://colab.research.google.com/github/akiroussama/PrivIMU/blob/main/notebooks/PrivIMU_MotionSense_Demo.ipynb

Cloud notebook that requires zero local installation. It (i) installs dependencies, (ii) downloads MotionSense, (iii) runs the full Random Forest training with 5-fold GroupKFold, and (iv) regenerates reports/metrics.json along with the three figures attached to this e-mail. End-to-end runtime ≈ 10 minutes on a free Colab CPU. This lets you independently verify that the numbers in the report are produced by the code, starting from a blank slate.
